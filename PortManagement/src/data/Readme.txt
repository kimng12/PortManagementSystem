The first line in each file serves as a header to indicate what each column represents.
For boolean values like Landing Ability, we can use true and false.
Dates can be stored in a standard format like YYYY-MM-DD HH:mm:ss.
Passwords in the Users.txt file are stored in plain text for simplicity, but in a real-world scenario, they should be hashed and salted for security reasons.
The structure can be further refined or expanded based on additional requirements or details.
